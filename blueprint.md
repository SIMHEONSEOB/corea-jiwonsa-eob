# 프로젝트 개요

이 프로젝트는 국가 지원 사업 정보를 웹 페이지에 표시합니다. 현대적인 웹 표준과 웹 컴포넌트를 활용하여 재사용 가능하고 관리하기 쉬운 UI를 구현합니다.

## 스타일, 디자인 및 기능

*   **반응형 디자인:** 다양한 화면 크기에 맞춰 레이아웃이 조정됩니다.
*   **카드형 UI:** 각 지원 사업 정보는 개별 카드 형태로 표시되어 시각적으로 구분됩니다.
*   **웹 컴포넌트 (`PolicyCard`):** `main.js`에 정의된 `PolicyCard`라는 커스텀 엘리먼트를 사용하여 지원 사업 카드를 생성합니다. 이는 재사용 가능하고 독립적인 UI 요소를 제공합니다.
*   **CSS 변수:** `:root`에 정의된 CSS 변수를 사용하여 색상, 그림자 등의 스타일을 중앙에서 관리하고 쉽게 변경할 수 있습니다.
*   **동적 데이터 로딩:** `main.js`에 정의된 `policyData` 배열에서 데이터를 가져와 웹 컴포넌트를 생성하고 `index.html`의 `.card-container`에 동적으로 추가합니다.

## 현재 변경 요청에 대한 계획 및 단계

**요청:** `main.js`의 데이터를 참고하여 `index.html`의 "주요 지원 사업" 칸에 넣어달라.

**설명:**
현재 프로젝트 설정에서 `index.html`의 "주요 지원 사업" 섹션은 `main.js` 파일에 의해 동적으로 채워지도록 이미 구현되어 있습니다.

1.  `index.html` 파일에는 `card-container`라는 클래스를 가진 `div` 요소가 있습니다. 이 요소가 지원 사업 카드들을 담는 컨테이너 역할을 합니다.
2.  `main.js` 파일에는 `PolicyCard`라는 웹 컴포넌트(Custom Element)가 정의되어 있습니다. 이 컴포넌트는 각 지원 사업 정보를 표시하는 카드 UI를 캡슐화합니다.
3.  `main.js` 파일의 `policyData` 배열에 실제 지원 사업 정보가 포함되어 있습니다.
4.  페이지 로드 시 `main.js` 스크립트가 `policyData` 배열을 반복하면서 각 데이터 항목에 대해 `policy-card` 웹 컴포넌트를 생성하고, `index.html`의 `card-container`에 추가합니다.

**결론:** 데이터는 이미 `main.js`에 의해 동적으로 `index.html`에 표시되도록 설정되어 있으므로, `index.html` 파일 자체를 직접 수정하여 데이터를 넣을 필요는 없습니다. 모든 코드는 현재 요청된 상태로 업데이트되었습니다.

만약 웹 페이지에서 데이터가 보이지 않는다면, 브라우저 캐시 문제이거나, 개발 서버가 제대로 실행되고 있지 않은 경우일 수 있습니다. 브라우저 캐시를 지우고 페이지를 새로고침하거나, 개발 서버를 재시작해 보세요.
