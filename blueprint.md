# 프로젝트 개요

이 프로젝트는 국가 지원 사업 정보를 웹 페이지에 표시합니다. 현대적인 웹 표준과 웹 컴포넌트를 활용하여 재사용 가능하고 관리하기 쉬운 UI를 구현합니다.

## 스타일, 디자인 및 기능

*   **반응형 디자인:** 다양한 화면 크기에 맞춰 레이아웃이 조정됩니다.
*   **카드형 UI:** 각 지원 사업 정보는 개별 카드 형태로 표시되어 시각적으로 구분됩니다.
*   **웹 컴포넌트 (`PolicyCard`):** `main.js`에 정의된 `PolicyCard`라는 커스텀 엘리먼트를 사용하여 지원 사업 카드를 생성합니다. 이는 재사용 가능하고 독립적인 UI 요소를 제공합니다.
    *   **상세 정보 확장 기능:** 각 카드에 `<details>` 태그를 활용한 "상세 정보 보기" 섹션을 추가하여, 사용자가 필요에 따라 더 많은 정보를 확인하고 숨길 수 있도록 했습니다.
*   **CSS 변수:** `:root`에 정의된 CSS 변수를 사용하여 색상, 그림자 등의 스타일을 중앙에서 관리하고 쉽게 변경할 수 있습니다.
*   **동적 데이터 로딩:** `main.js`에 정의된 `policyData` 배열에서 데이터를 가져와 웹 컴포넌트를 생성하고 `index.html`의 `.card-container`에 동적으로 추가합니다.

## 현재 변경 요청에 대한 계획 및 단계

**요청:** "예비창업패키지"에 대한 상세 데이터를 웹 페이지에 표시해달라. (카드 표시 문제 및 CSP 오류 해결)

**설명:**
이전 요청에서 "예비창업패키지" 카드가 제대로 표시되지 않는 문제가 발생했으며, "Content Security Policy (CSP) of your site blocks the use of 'eval' in JavaScript" 오류가 보고되었습니다.

**문제 원인 분석 및 해결:**

1.  **카드 표시 문제 (1차 해결):**
    *   **원인:** `main.js`의 `PolicyCard` 웹 컴포넌트 내부에서 HTML 템플릿 (`template.innerHTML`) 생성 로직에 오류가 있었습니다. 상세 정보 (`detailsHtml`)가 메인 카드 (`cardDiv`)의 Shadow DOM에 올바르게 포함되지 않아 카드가 예상대로 렌더링되지 않았습니다.
    *   **해결:** `main.js` 파일을 수정하여, `PolicyCard` 웹 컴포넌트의 `constructor` 내부에서 `detailsData`를 먼저 파싱하고, 이를 바탕으로 `detailsHtml` 문자열을 생성한 다음, 이 `detailsHtml`을 메인 카드 템플릿 (`template.innerHTML`) 내에 직접 삽입하도록 변경했습니다. 이를 통해 웹 컴포넌트가 Shadow DOM을 생성할 때 기본 카드 내용과 상세 정보 섹션이 한 번에 올바르게 구성되도록 했습니다.

2.  **CSP 오류 문제 (2차 해결):**
    *   **원인:** 웹 컴포넌트 내에서 동적으로 HTML 문자열을 생성하고 `innerHTML`을 사용하는 방식이 Firebase Studio 환경의 엄격한 Content Security Policy에 의해 `eval()` 사용으로 감지되어 차단되었습니다. `JSON.parse` 자체는 문제가 없으나, `innerHTML`을 통한 동적인 DOM 생성이 CSP에 저촉된 것으로 보입니다.
    *   **해결:** `main.js` 파일의 `PolicyCard` 웹 컴포넌트 코드를 다시 수정했습니다. HTML 문자열을 생성하여 `innerHTML`에 할당하는 대신, `document.createElement()` 및 `appendChild()` 메서드를 사용하여 모든 DOM 요소를 직접 생성하고 조작하도록 변경했습니다. 이 방법은 어떠한 문자열도 JavaScript 코드로 "평가"하지 않으므로, 가장 엄격한 CSP 환경에서도 안전하게 작동하며 오류를 해결할 수 있습니다.

**결론:** 이제 웹 페이지에서 "예비창업패키지" 카드가 제대로 표시되고, 카드 안의 "상세 정보 보기"를 클릭하면 제공된 상세 정보가 나타날 것으로 예상됩니다. CSP 오류도 해결되었어야 합니다.

**확인 지침:**
*   브라우저 캐시를 지우고 페이지를 새로고침하거나 개발 서버를 재시작해 보세요.
*   웹 페이지에서 "예비창업패키지" 카드가 나타나는지 확인하고, "상세 정보 보기" 버튼을 클릭하여 상세 정보가 올바르게 표시되는지 확인해 주세요.
*   **브라우저의 개발자 도구(Console 탭)를 열어, 이전에 발생했던 CSP 관련 오류 메시지가 더 이상 나타나지 않는지 확인해 주세요.** 여전히 오류가 발생한다면 알려주시면 더욱 정확하게 진단하고 해결하겠습니다.
